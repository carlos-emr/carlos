## 2024-05-13 - Static Regex Pattern Compilation & ReDoS Avoidance
**Learning:** In the CARLOS EMR codebase, repetitive dynamic `java.util.regex.Pattern.compile()` calls inside heavily used loops (e.g., in parsing condition expressions like `DSValue.java`) cause performance bottlenecks and unnecessary object allocations. Furthermore, unanchored greedy or dangerous wildcard regexes like `'.+?'` can trigger catastrophic backtracking and ReDoS SonarCloud warnings. Redundant capturing groups like `([^\s]+$)` also slow down the regex engine without any functional benefit.
**Action:** Always refactor repetitive `Pattern.compile()` calls to `private static final Pattern` fields for efficient static reuse. When optimizing the patterns themselves, replace dangerous wildcards with negated character classes (e.g., change `'.+?'` to `'[^']+'`) to prevent ReDoS, and remove redundant capturing groups (e.g., `([^\s]+$)` to `[^\s]+$`) to simplify engine execution.
