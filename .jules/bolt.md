## 2024-05-18 - Pre-compiled Regex Patterns and Avoidance of Greedy Quantifiers
**Learning:** Repetitive inline compilation of Java regex patterns (via `Pattern.compile()` or inline regex methods like `String.replaceAll()`) can introduce measurable overhead, especially when parsing multiple strings or expressions. Unanchored greedy quantifiers (like `.+?`) combined with the `Pattern` matcher can cause slow execution and trigger ReDoS alerts.
**Action:** When extracting data using patterns, define regex as `private static final Pattern` constants. Furthermore, replace greedy unanchored quantifiers with negated character classes (e.g. `[^']+` instead of `.+?`) and simplify capture groups (e.g. `[^\s]+$` instead of `([^\s]+$)`) to avoid backtracking and improve engine execution performance.
