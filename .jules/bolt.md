## 2026-04-14 - Pre-compile Regex Patterns in Loop/Static Contexts
**Learning:** In the `DSValue` parsing logic, Java's `Pattern.compile()` was being invoked repeatedly within static methods (`createDSValues`, `createDSValue`, `indexOfNotQuoted`). Because regex compilation is a relatively expensive operation that creates a state machine under the hood, calling `Pattern.compile()` dynamically within methods that parse individual rules or guidelines creates unnecessary overhead and degrades performance, especially at scale.
**Action:** When a regex pattern is completely static, always extract it to a `private static final Pattern` field at the class level. This ensures the regex is compiled only once when the class is loaded, rather than being recompiled every time the method is executed.
