## 2026-04-19 - Safe Pattern Pre-compilation
**Learning:** When attempting to extract identical-looking local string arrays (like `durationUnits` for regex compilation) into a shared `static final` constant, you must verify the actual string values are identical. In `RxUtil.java`, `durationUnits` in `isMitte` included start-of-line anchors and prefix matchers (`^[0-9]+`), while `durationUnits` in `instrucParser` did not, because it appended digits dynamically later in the loop. Unintentionally merging them caused a double-digit prefix regression (`[0-9]+[0-9]+`) that broke single-digit parsing.
**Action:** Do not assume identically named local arrays in legacy code are actually identical. Verify the contents and their downstream usage (especially dynamic concatenation) before refactoring them into a shared global constant.
