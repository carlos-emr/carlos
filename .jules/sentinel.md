## 2025-02-14 - Fix SQL injection in EFormReportToolDaoImpl dynamically assembled INSERT queries
**Vulnerability:** Found a SQL injection in `EFormReportToolDaoImpl.populateReportTableItem` where the native SQL `INSERT` statement for an eForm was dynamically assembled. Specifically, the data values from `EFormValue` (`v.getVarValue()`) were concatenated directly into the `VALUES` clause of the query string using string concatenation, which poses a severe risk of SQL injection since `varValue` is usually derived from user input.
**Learning:** When building dynamic SQL statements where the number of values is dynamic (like based on dynamic eForm data), you might still need to dynamically assemble the structure (like `?`, `?1`, etc). However, the values themselves must ALWAYS be parameterized.
**Prevention:** To prevent SQL injection in dynamically assembled statements, append parameter placeholders in a loop for the values clause and bind them sequentially using `setParameter(index, value)` before executing the query instead of inserting the values as text.
