## 2026-10-10 - SQL Injection via dynamically concatenated INSERT statement in EFormReportToolDaoImpl
**Vulnerability:** Found a severe SQL injection vulnerability in `EFormReportToolDaoImpl.populateReportTableItem` where values mapped from an `EFormValue` representation were being dynamically appended to a raw insert statement without proper parameter binding.
**Learning:** Even if the query statement uses some internal variables (such as ID, user demographics, dates, or providers), injecting raw values from an object containing dynamic variables can easily lead to SQL injection vulnerabilities. Dynamic SQL constructs in older parts of the system must carefully validate the variables that they iterate over and replace string concatenation with parameter bindings wherever possible.
**Prevention:** Avoid dynamically concatenating objects with `VALUES (...)` statements by adopting `Query#setParameter(index, value)` properly. When building dynamic parameter sequences, iterate using a running loop counter to inject ordered placeholders `?n` and then correctly bind variables to the correct loop index securely.
