## 2024-05-24 - Command Line Password Exposure in Runtime exec
**Vulnerability:** The database password was being passed as a command-line argument (`--password=...`) to `mysqldump` via `Runtime.getRuntime().exec()`. This exposes the plaintext password to any user on the system who runs `ps` or top-like utilities while the command is executing. It also triggers Semgrep PRO command injection alerts due to building commands via array assignment.
**Learning:** External system commands using `Runtime.getRuntime().exec` or `ProcessBuilder` that handle sensitive information should never accept secrets directly via process arguments, as process arguments are globally readable in process monitoring tools.
**Prevention:** Always use `ProcessBuilder` for spawning external processes and pass arguments directly to the constructor (to prevent command injection warnings). Inject secrets securely via the process's environment variables (e.g., `pb.environment().put("MYSQL_PWD", password)`) since environment variables are typically restricted to the running user and not broadcast via standard process listing tools.
