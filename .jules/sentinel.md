## 2026-04-17 - Prevent SQL Injection via Dynamic EForm Report Tool Schema Creation
**Vulnerability:** SQL injection could happen in `EFormReportToolDaoImpl` during dynamic table creation and population, because dynamic string values (`field` inside `addNew` and `varName` inside `populateReportTableItem`) from eForm configurations could contain SQL syntax. Also values and parameters were constructed using raw string concatenation instead of parameterized binding.
**Learning:** JPA native queries (`createNativeQuery`) do not allow column names to be parameterized safely. However, we also can't easily allowlist due to potential user-entered formatting logic like dashes or numbers.
**Prevention:** Use a targeted blocklist against malicious SQL characters (like backticks \` which wrap MySQL identifiers) on the column headers before concatenating them. Ensure that data values (like form variables and dates) are fully converted to parameterized bound variables instead of raw string concatenation.
