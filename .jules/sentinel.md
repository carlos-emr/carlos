## 2025-02-28 - Command Injection risk by passing password in process arguments
**Vulnerability:** Passing a database password as part of the string arguments given to `Runtime.getRuntime().exec` exposes the password to command-line argument viewers such as `ps`. Furthermore, constructing commands as an array containing flags concatenated directly with their corresponding variables could easily cause Command Injection and leads to false positives for tools like SpotBugs/Semgrep.
**Learning:** Legacy backup features, like `AuditLogManager.purgeAuditLog`, frequently use runtime executable features incorrectly. By treating command pieces as one whole chunk, you introduce both vulnerability risks and system instability.
**Prevention:** Always use `ProcessBuilder` instead of `Runtime.getRuntime().exec`. When you need to inject sensitive arguments such as passwords, place them as an environment variable by using `pb.environment().put("MYSQL_PWD", password);`. In addition, to prevent injection and analysis false positives, separate the flag and value as separate arguments instead of joining them (e.g., separate `--user` and `user`).
