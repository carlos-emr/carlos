## 2024-05-15 - SQL Injection via Native Query Parameter Concatenation
**Vulnerability:** The `EFormReportToolDaoImpl` class contained severe SQL injection vulnerabilities in `populateReportTableItem` and `markLatest` methods where user-provided inputs (`EFormValue.getVarValue()` and integer values like `demoNo`) were concatenated directly into strings fed to `entityManager.createNativeQuery()`.
**Learning:** JPA native queries can still be vulnerable to SQL injection if input is concatenated rather than parameterized. Using strings formatted directly as SQL allows malicious string escape to execute arbitrary SQL commands on the database, bypassing the ORM's safety features.
**Prevention:** Always parameterize inputs in native queries (using `?1`, `?2`, or `:name`) even when dynamic columns/table names are involved. Iterate through user inputs (like lists of values) and append safe `?N` parameter placeholders to the SQL string, binding the respective values later via `Query.setParameter(N, value)`.
