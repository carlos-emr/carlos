## 2026-04-12 - SQL Injection in EFormReportToolDaoImpl
**Vulnerability:** SQL Injection in dynamic `INSERT`, `UPDATE`, and `SELECT` queries constructed via string concatenation of user-controlled variables. In `EFormReportToolDaoImpl.java`, `populateReportTableItem` manually constructed an `INSERT INTO` native query using `sb.append` without any parameterization for `fdid`, `demographicNo`, `dateFormCreated`, `providerNo`, and `EFormValue.varValue`. Similarly, `markLatest` concatenated variables like `demoNo` and `idList.get(0)`.
**Learning:** Legacy repository classes manually assembling native SQL or JPA strings with user data instead of relying on parameterized execution pose a critical risk, even when part of the query (like dynamic table names) must be concatenated. In Java, parameterized variables cannot be used for dynamic table/column names, but MUST always be used for query values to avoid SQL injection.
**Prevention:** Use prepared statement place holders `?` or named parameters `:paramName` for all variable inputs. For native queries building dynamic value lists, iterate and append `?` placeholders, and then bind them sequentially using `query.setParameter()`.
