## 2026-04-25 - Prevent SQL Injection in Dynamic Table Creation
**Vulnerability:** The `addNew` method in `EFormReportToolDaoImpl` was dynamically concatenating untrusted column names from the `fields` list into a `CREATE TABLE` statement without validation, allowing SQL injection via backticks. Additionally, existing `providerNo` test expectations were outdated, asserting string literals instead of the actual `?4` parameterization used in the source.
**Learning:** Overly strict allowlists (like `^[a-zA-Z0-9_]+$`) for dynamic column names (especially in eForms) can cause functional regressions since input names often contain hyphens. When the system wraps dynamic column names in backticks, failing to sanitize them can lead to SQL injection. Also, legacy tests may assert outdated string-concatenated formats, resulting in test failures even when the source code is already safely parameterized.
**Prevention:** Apply a targeted blocklist validation (e.g., rejecting backticks `.*[\`].*`) to dynamic column names prior to string concatenation. Keep tests up to date with parameterized implementations, using `verify(mockQuery).setParameter()` instead of checking for hardcoded parameter values in SQL strings.
