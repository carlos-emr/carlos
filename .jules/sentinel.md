## 2026-04-15 - Prevent SQL Injection in dynamically constructed native queries
**Vulnerability:** A native query was constructed with `parameter` coming from variable (potentially untrusted context). It was passed down and dynamically added into `select special,special_instruction from drugs where " + parameter + " = '" + value ...`. This creates a vulnerability where `parameter` could be a SQL injection vector.
**Learning:** Native queries must validate dynamic column names. When variables are concatenated to create the query structure (like a column name or a part of the table logic) rather than passed via parameter binding (`?1` or `:name`), they need strict validation, such as checking against an allowlist or a regex like `^[a-zA-Z0-9_]+$`, prior to concatenation. Values must still be parameterized.
**Prevention:** Always ensure dynamic schema changes (tables, columns) in queries are stringently validated via an allowlist or regex, and values are securely parametrized via JPA mechanisms.
