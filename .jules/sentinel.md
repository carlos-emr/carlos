## 2024-04-14 - Fix SQL Injection in FrmForm2Action
**Vulnerability:** A SQL injection vulnerability was found in `src/main/java/io/github/carlos_emr/carlos/form/pageUtil/FrmForm2Action.java` where the `demographicNo` was directly concatenated into the SQL string `String sql = "SELECT * FROM form" + formName + " WHERE demographic_no='" + demographicNo + "' AND ID=0";` before passing to `saveFormRecord()`.
**Learning:** Legacy codebase or unreviewed pull requests sometimes concatenate query strings using single quotes. Even if input comes from previously populated beans, standard parameterization must always be utilized to ensure that single quote characters inside inputs do not accidentally or maliciously break syntax and inject logic.
**Prevention:** Always use parameterized SQL placeholders (`?` or `:name`) when appending parameters to native queries or HQL queries. `FrmRecordHelp`'s `saveFormRecord()` has an overloaded method taking an `Object... params` array to accommodate this.
