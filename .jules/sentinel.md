## 2026-04-25 - Prevent SQL Injection in Dynamic Table Creation
**Vulnerability:** The `addNew` method in `EFormReportToolDaoImpl` was dynamically concatenating untrusted column names from the `fields` list into a `CREATE TABLE` statement without validation, allowing SQL injection via backticks. Additionally, existing `providerNo` test expectations were outdated, asserting string literals instead of the actual `?4` parameterization used in the source.
**Learning:** Overly strict allowlists (like `^[a-zA-Z0-9_]+$`) for dynamic column names (especially in eForms) can cause functional regressions since input names often contain hyphens. When the system wraps dynamic column names in backticks, failing to sanitize them can lead to SQL injection. Also, legacy tests may assert outdated string-concatenated formats, resulting in test failures even when the source code is already safely parameterized.
**Prevention:** Apply a targeted blocklist validation (e.g., rejecting backticks `.*[\`].*`) to dynamic column names prior to string concatenation. Keep tests up to date with parameterized implementations, using `verify(mockQuery).setParameter()` instead of checking for hardcoded parameter values in SQL strings.

## 2026-04-25 - Prevent OWASP Encoder Null-to-String Cast Vulnerabilities
**Vulnerability:** In multiple JSPs (e.g., `generatePreviewPDF.jsp`, `attachmentFrameset.jsp`), the standard `org.owasp.encoder.Encode` functions were used directly instead of the project's safe wrapper. Specifically, standard OWASP encoders render literal `"null"` when passed a null value, leading to confusing artifacts and potential downstream processing errors when handling uninitialized dynamic inputs.
**Learning:** Using `Encode.forXxx` directly in JSPs or EL bypasses the project's null-safety wrapper (`SafeEncode` / `carlos:encode`), which handles null inputs defensively (returning empty strings instead of `"null"`). CI linting explicitly checks for this and fails the build.
**Prevention:** Strictly enforce the use of `io.github.carlos_emr.carlos.utility.SafeEncode` in scriptlets and the `<%@ taglib uri="/WEB-INF/carlos.tld" prefix="carlos" %>` with `${carlos:forXxx}` for Expression Language. Never use `Encode.forXxx` or `${e:forXxx}` directly in JSPs.

## 2026-04-25 - MariaDB DevContainer Init Script Permissions
**Vulnerability:** DevContainer DB initialization (`populate_db.sh`) attempts to create the eForm images directory (`mkdir -p /var/lib/OscarDocument/...`). Because the init script runs as the `mysql` user within the `mariadb:10.5` container, creating directories in the root-owned `/var/lib` fails with `Permission denied`.
**Learning:** Container initialization scripts executing as restricted users cannot write to root-owned default volumes/mounts without prior configuration.
**Prevention:** Pre-create the necessary directory paths in the `Dockerfile` running as root and correctly set the ownership (`chown -R mysql:mysql /var/lib/OscarDocument`) prior to runtime initialization.
