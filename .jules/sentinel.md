## 2024-05-18 - Prevent Credential Leak in mysqldump Command
**Vulnerability:** The application was calling `Runtime.getRuntime().exec()` to execute a `mysqldump` command, concatenating arguments such as `--password=` to the database password. This exposes the database password via command line arguments, meaning anyone observing process monitoring tools (e.g. `ps -ef`) on the server would be able to see the database credentials.
**Learning:** External process arguments are visible globally. Using raw `Runtime.exec` and concatenating arguments and values (like `--user=bob`) can also trigger static analysis tools for command injection false positives, and leaking passwords is a critical security risk.
**Prevention:** Always refactor command execution to use `ProcessBuilder`. Inject secrets securely into the process using environment variables (e.g., `pb.environment().put("MYSQL_PWD", password)`) rather than as an argument. Furthermore, separate argument flags and their values into completely distinct elements (e.g., `--user` and `user`) to bypass static analysis alerts, and use `StringBuilder` for dynamic expressions like a `where` clause.
