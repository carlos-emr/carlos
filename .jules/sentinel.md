## 2026-04-25 - Prevent SQL Injection in Hl7LinkDao
**Vulnerability:** SQL injection vulnerabilities existed in `Hl7LinkDao.java` because user inputs (`provider_no`, `orderby`, `start`, and `end` dates) were concatenated directly into the `findReports` native SQL query. The `provider_no` placeholder (`@provider_no`) was wrapped in quotes within the query.
**Learning:** Using an ORM does not automatically protect against SQL injection when constructing dynamic native SQL queries. When attempting to parameterize a dynamic query using `?` or named placeholders (`:`), if the original query contained the placeholder within quotes (e.g., `'@provider_no'`), simply replacing the placeholder name with the bound variable (`:provider_no`) will result in the string literal `":provider_no"` and the database driver failing to locate the binding, causing a query execution error (`IllegalArgumentException: Parameter with that name did not exist`).
**Prevention:** Always use parameterized placeholders for bind variables and ensure they are not enclosed in quotes within the SQL statement. For `ORDER BY` clauses that cannot be parameterized by the database driver, use an explicit allowlist mapping of expected input strings to safe, hard-coded column names.
